# RAG 데이터 디렉토리 구조

## 디렉토리 명명 규칙

### 표준 디렉토리 구조 (대문자)

- `K0_intake/` - 초기 상담 메시지
- `K1_case_type/` - 사건 유형 분류 기준
- `K2_required_fields/` - 필수 필드 및 질문 템플릿
- `K3_risk_rules/` - 위험도 판단 규칙
- `K4_output_format/` - 출력 포맷 템플릿
- `common/facts/` - 공통 사실 패턴

### 구버전 디렉토리 (소문자) - 제거 예정

- `k1/` - 구버전 형식, `K1_case_type/`로 통합 예정
- `k2/` - 구버전 형식, `K2_required_fields/`로 통합 예정
- `k3/` - 구버전 형식, `K3_risk_rules/`로 통합 예정
- `k4/` - 구버전 형식, `K4_output_format/`로 통합 예정

**주의**: 소문자 디렉토리의 파일들은 구버전 형식이며, 향후 제거될 예정입니다. 새로운 파일은 대문자 디렉토리에 추가하세요.

## 디렉토리 구조 통일 계획

1. 소문자 디렉토리(`k1/`, `k2/`, `k3/`, `k4/`)의 파일 검토
2. 중복 파일 확인 및 제거
3. 필요한 파일은 대문자 디렉토리로 이동
4. 소문자 디렉토리 삭제
